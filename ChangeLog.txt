version 2.2.5 (4/19/2013)
--------------------------------------------------------------------------------
    * Add a default PPM value to the .NET library
    * Fix a bug where the channel wasn't being set correctly (causing FCR,
      including clock operations to not work on 2nd channel)
    * Lots of small performance improvements when handling incoming frames
    * Increase timeout duration to decrease the frequency of timeout errors at 
      lower data rates
    * Fix a bug where stream data wasn't terminated on memory constraint

version 2.2.4 (4/10/2013)
--------------------------------------------------------------------------------
    * Make the CMDR register write only in the .NET library
    * Bug fix regression where memory usage wasn't being calculated correctly
      (causing runaway memory use)
    * Add initial support for memory-mapped firmware
    * Miscellaneous little bug fixes

version 2.2.3 (3/25/2013)
--------------------------------------------------------------------------------
    * Bug fix random crash when using a framing based mode
    * Bug fix in driver where 'append status' was being ignored (full fix)

version 2.2.2 (3/13/2013)
--------------------------------------------------------------------------------
    * Bug fix in .NET library where extra data was being returned from the 
      Read function
    * Bug fix in driver where 'append status' was being ignored
    * Bug fix in .NET library where registers weren't being set correctly
    * Update SerialFC to v2.1.2 
      (https://github.com/commtech/serialfc-windows/blob/master/ChangeLog.txt)

version 2.2.1 (2/25/2013)
--------------------------------------------------------------------------------
    * Performance improvements
    * Add Python support
    * Update SerialFC to v2.1.1 
      (https://github.com/commtech/serialfc-windows/blob/master/ChangeLog.txt)

version 2.2.0 (2/1/2013)
--------------------------------------------------------------------------------
    * Update SerialFC to v2.1.0 
      (https://github.com/commtech/serialfc-windows/blob/master/ChangeLog.txt)
    * Libraries are now built for XP support
    * fscc_read_with_timeout now uses CancelIo instead of CancelIoEx to support
      XP.

version 2.1.0 (1/11/2013)
--------------------------------------------------------------------------------
    * Add asynchronous support

version 2.0.2 (11/16/2012)
--------------------------------------------------------------------------------
    * Improve port location naming in the Device Manager
    * Fix a bug in fscc_read_with_timeout where a frame wasn't cancelled upon 
      timeout
    * Fix a bug where X-Sync mode wasn't considered stream data if there wasn't 
      any framing
    * Improved debug prints
    * IRQ and memory optimizations
    * Fix a data loss bug in transparent mode

version 2.0.1 (11/05/2012)
--------------------------------------------------------------------------------
    * Bug fix that prevented initiating a read before data had arrived
    * Improve debug prints
    * Changed each port's priority level to SERIAL_PORT
    * Miscellaneous BSOD bug fixes
    * Miscellaneous data loss bug fixes
    * Change SDDL permissions to not require administrator privileges
    * Add more explicit exceptions to the C++ library
    * Fix a C++ library connection bug 
    * Fix a generic library bug preventing setting the clock frequency
